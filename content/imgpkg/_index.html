Bundle your bits with guaranteed immutability

Package, distribute, and relocate your container images and their dependent configuration files as if they were a single OCI image: a bundle. Consume bundles with confidence that their contents are unchanged after relocation.

See Examples Download Binary

OCI Registry-compliant

Imgpkg packages your files into images using OCI image specifications, allowing the image to be pushed to any registry (like Docker, Harbor, ECR, GCR, …).

Immutable references

Images are identified with a unique SHA256 digest based on the file contents. Imgpkg uses that digest to ensure that the relocated files are identical to those originally pushed.

Unified distribution

A bundle captures your configuration files and a list of references to images on which they depend. Copying your bundle will relocate your configuration files along with the referenced images to your destination (i.e. repository, tarball). Imgpkg treats the whole thing as one.

Features

Air-gapped environments

Copy a bundle into a compressed tarball for offline relocation, then push to a private registry for true air-gapped file transfers.

Lock file in/output

Produces a lock file that can later ensure exactly the same image versions (with digests) are used.

Content agnostic

Imgpkg makes no assumptions about the contents of bundles. It can manage all kinds of container+configuration files: Kubernetes deployments, Machine Learning models, etc.

Basic Usage

{{< highlight bash "" >}} # Create a bundle — in a registry — with configuration and image references $ imgpkg push --file=bundle-dir/ --bundle=example/bundle-repo-name # Copy a bundle (w/ dependency images) to another registry $ imgpkg copy --bundle=example/bundle-repo-name \ --to-repo=internal-registry/bundle-repo-name # Download bundle into a tarball for air-gapped relocation $ imgpkg copy --bundle=internal-registry/bundle-repo-name \ --to-tar=/Volumes/secure-thumb/bundle.tar # Pull a (relocated) bundle to local machine $ imgpkg pull --bundle=internal-registry/bundle-repo-name \ --output=/tmp/bundle-dir {{< / highlight >}}
